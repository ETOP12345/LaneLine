# Lane Line

Lane Line is a swim time checker for swimmer profiles, best times, PNS cuts, USA Swimming motivational standards, meet qualification tracking, and swimming-video stroke analysis.

## Run

```bash
npm install
npm run dev
```

Open `http://localhost:5180`.

The app uses the included Node server for the static page, USA Swimming best-times refresh endpoints, Swimcloud best-times proxy, and the video-analysis API.

## Video analysis

The **Video analysis** tab uploads a short MP4, MOV, or WebM swimming clip to the local Node server. An optional focus note can identify the target by lane, cap color, or visible position. The server:

- validates file type and size before processing
- inspects real video metadata with `ffprobe`
- rejects clips that are too long or below the minimum resolution
- extracts dense candidate frames with `ffmpeg` at 6 frames per second by default
- selects up to 30 representative timestamped frames
- sends selected frames to the OpenAI Responses API with `gpt-5.6-terra` when `OPENAI_API_KEY` is configured
- returns a demo result when no OpenAI key is configured so the UI flow can be exercised without API spend
- deletes temporary local files after processing

Default free-tier-friendly limits are 30 analyses per day, 10 per hour, 30 seconds per clip, and 50 MB per upload.

## Environment

Copy `.env.example` to `.env` for local secrets and tunable limits:

```bash
cp .env.example .env
```

At minimum, real AI analysis requires these server-side values:

```text
OPENAI_API_KEY=
OPENAI_VISION_MODEL=gpt-5.6-terra
APP_ACCESS_CODE=choose-a-shared-site-code
SESSION_SECRET=choose-a-long-random-value
```

Never put the OpenAI API key in browser JavaScript.

## Deploy on Render Free

The repository includes a Docker image and `render.yaml` Blueprint. The image installs FFmpeg, runs the Node server as a non-root user, and listens on Render's `PORT` automatically.

1. Push this repository to GitHub.
2. In Render, create a **Blueprint** and connect this repository.
3. Enter `OPENAI_API_KEY` and `APP_ACCESS_CODE` when Render asks for secret values. Do not commit either value.
4. Apply the Blueprint and wait for `/healthz` to become healthy.

`APP_ACCESS_CODE` is a shared password for the small group using the analysis endpoint. `SESSION_SECRET` is generated by Render. The free web service can spin down after inactivity, so the first request after a quiet period may take about a minute. Its filesystem is ephemeral; uploads and extracted frames are intentionally temporary.

The website hosting can use Render's free allowance, but OpenAI API inference is billed separately to the OpenAI API account associated with the server-side key.

## FFmpeg requirement

Install FFmpeg locally or on the hosting service so both commands are available on `PATH`:

```bash
ffmpeg -version
ffprobe -version
```

## Files

- `index.html` - Lane Line interface, swim standards logic, and video-analysis UI
- `server.js` - local static server, swim-times proxies, and video-analysis route mounting
- `videoAnalysis.js` - upload validation, rate limiting, FFprobe/FFmpeg processing, frame selection, and OpenAI integration
- `test/videoAnalysis.test.js` - Node tests for validation and frame selection
- `package.json` - run, test, and check scripts
- `.env.example` - server-side configuration template
- `Dockerfile` - production Node + FFmpeg image
- `render.yaml` - Render Free Blueprint and secret declarations
